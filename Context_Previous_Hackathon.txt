MediGraph
Intelligent Document Parsing & Knowledge Graph for Healthcare Facility Intelligence
Built for the Bridging Medical Deserts track (Virtue Foundation / Databricks). MediGraph turns messy facility CSV data into a queryable knowledge graph and an agentic AI layer—so planners can ask questions in plain language, see where care exists, and identify gaps and medical deserts.
What it does
* Upload facility CSVs (e.g. Virtue Foundation Ghana format); optionally use LLM to extract entities from free-text columns (equipment, procedures, specialties, capabilities).
* Build a Neo4j knowledge graph with facilities, locations, equipment, procedures, and typed relationships.
* Query in natural language: “Which facilities lack ultrasound?”, “List facilities in Accra with their specialties”—get answers with citations and optional reasoning trace.
* Plan via a dedicated Planning tab (resource allocation, distribution, capacity questions).
* Visualize the graph and a map of locations; run built-in medical-deserts and equipment-gaps analysis.

VirtueCommand
AI-Powered Health Facility Intelligence for Ghana
Overview
VirtueCommand turns messy, unverified health facility data into a structured medical knowledge graph, then exposes it through a multi-agent AI system that answers operational questions for NGO mission planners. From "which facilities offer dialysis in Northern Region?" to "where should we deploy a surgical mission team?", the system retrieves graph data, enriches it with demographic context, verifies claims through structured debate, and produces scored deployment recommendations.
External Data Sources
• Virtue Foundation FDR: 987 facility records from web scraping + LLM extraction (websites, social media, health registries). Includes contact, address, specialties, procedures, equipment, capabilities, and raw free-text.
• DHS Health Indicators: DHS 2022 subnational data for all 16 Ghana regions — child/maternal mortality, anemia, vaccination rates, insurance coverage, facility delivery rates.
• Population & Accessibility: Population density, travel time/road quality, OpenStreetMap facility boundaries.
• WHO Health Systems: National-level health system rankings for country context.
Knowledge Graph Pipeline
Raw CSV data is transformed into a knowledge graph (1,019 nodes, 7,794 edges) through an 8-stage pipeline:
Stage Process Output
1. Load & Clean Parse JSON-encoded list fields, normalize nulls 987 cleaned rows
2. Region Normalize 54+ raw address variants → 16 canonical regions Standardized geography
3. Deduplicate Merge multi-source rows by pk_unique_id 742 unique facilities
4. Vocab Normalize Regex + LLM batch (GPT-4o-mini) mapping 35 capabilities, 48 equipment types
5. Build Graph Create typed nodes + edges with confidence scores 6 node types, 8 edge types
6. Inference Derive LACKS and COULD_SUPPORT edges Gap + upgrade-opportunity signals
7. Desert Detection Flag regions with zero providers for a specialty DESERT_FOR edges
8. Export Serialize to pickle + GraphML + metadata JSON Production-ready graph
Key innovation: Treating absence as signal — missing edges in the knowledge graph surface gaps, anomalies, and upgrade opportunities that flat data cannot reveal.
Multi-Agent Architecture
Supervisor pattern: one orchestrator delegates to specialist agents via tool calls. Each agent has a scoped tool set and system prompt. No handoffs — all coordination flows through the Supervisor.
Agent Role Tools Workflow
Supervisor Orchestrator — reads intent, delegates, synthesizes ask_analyst, ask_planner, ask_verifier, ask_rag_agent, run_facility_debate, run_mission_debate Routes: lookups → Analyst; planning → Analyst → Planner; trust → Verifier; docs → RAG
Analyst Graph data retrieval specialist resolve_terms, explore_overview, search_facilities, find_facility, count_facilities, search_raw_text, Vocabulary → Landscape VirtueCommand
